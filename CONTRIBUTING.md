# Contributing to Dynamic Weather Patterns

Thank you for your interest in contributing to the Dynamic Weather Patterns module! This document provides guidelines and information for contributors.

## Ways to Contribute

### Reporting Bugs

- Check existing issues first to avoid duplicates
- Use the bug report template
- Include:
  - Foundry VTT version
  - Module version
  - System being used
  - Steps to reproduce
  - Expected vs actual behavior
  - Console errors (F12 → Console tab)
  - Screenshots if applicable

### Suggesting Features

- Check existing feature requests first
- Use the feature request template
- Describe:
  - The problem you're trying to solve
  - Your proposed solution
  - Alternative solutions considered
  - How it benefits other users

### Contributing Code

- Fork the repository
- Create a feature branch (`git checkout -b feature/amazing-feature`)
- Make your changes
- Test thoroughly
- Commit with clear messages
- Push to your fork
- Open a Pull Request

### Improving Documentation

- Fix typos, clarify instructions
- Add examples and use cases
- Translate documentation
- Improve API documentation

## Development Setup

### Prerequisites

- Node.js 16+ (required for building)
- npm (comes with Node.js)
- Git
- Foundry VTT 11+ (for testing)
- A code editor (VS Code recommended)
- Basic knowledge of Vue 3 (Composition API) helpful for UI changes
- Familiarity with ES6+ JavaScript

### Local Development

1. Clone your fork:

   ```bash
   git clone https://github.com/yourusername/dynamic-weather.git
   ```

2. Install dependencies:

   ```bash
   cd dynamic-weather
   npm install
   ```

3. Build the module:

   ```bash
   npm run build
   ```

4. Link to Foundry modules directory (use dist folder):

   ```bash
   # Windows (using included batch file)
   npm run link-dev-dist

   # Or manually:
   mklink /J "%localappdata%\FoundryVTT\Data\modules\dynamic-weather" "path\to\your\clone\dist"

   # Linux/Mac
   ln -s /path/to/your/clone/dist ~/.local/share/FoundryVTT/Data/modules/dynamic-weather
   ```

5. Enable the module in Foundry VTT

6. Make changes in `src/` and rebuild with `npm run build`

### Development Scripts

- `npm run dev` - Start Vite dev server (port 30002) with hot module replacement
- `npm run build` - Build for production to `dist/`
- `npm run preview` - Preview production build
- `npm run build:db` - Build compendium packs from source JSON
- `npm run build:json` - Unpack compendium to source JSON
- `npm run link-dev` - Link project root to Foundry (for dev server)
- `npm run link-dev-dist` - Link dist folder to Foundry (for production build)
- `npm run unlink-dev` - Remove symbolic link

### Testing

- Test in a clean world with minimal modules
- Test with popular modules for compatibility
- Test all settings combinations
- Test with different rolltables
- Check browser console for errors
- Verify state persistence (reload browser)

### Code Style

- Use ES6+ features (ES modules, arrow functions, etc.)
- Use meaningful variable and function names
- Add JSDoc comments for public APIs
- Keep functions focused and small
- Use `async/await` for asynchronous code
- Follow existing code style
- **Vue Components**: Use Composition API with `<script setup>` syntax
- **Reactive Data**: Use Vue's `ref()` and `computed()` for reactivity
- **Lifecycle Hooks**: Use `onMounted()`, `onUnmounted()` for component lifecycle

## Pull Request Process

1. **Update Documentation**: If you change functionality, update the README and API docs

2. **Update Changelog**: Add your changes to CHANGELOG.md under "Unreleased"

3. **Test Thoroughly**: Ensure all features work and no existing features break

4. **Clear Commits**: Use descriptive commit messages following this pattern:

   ```
   feat: Add weather forecast feature
   fix: Correct transition calculation bug
   docs: Update API documentation
   refactor: Simplify rolltable parsing
   test: Add tests for compatibility calculation
   ```

5. **PR Description**: Include:
   - What changes were made
   - Why the changes were needed
   - How to test the changes
   - Screenshots/videos if applicable

6. **Respond to Feedback**: Be responsive to review comments and questions

## Coding Guidelines

### JavaScript Best Practices

```javascript
// Good: Clear, documented function
/**
 * Calculate weather compatibility score
 * @param {WeatherPattern} pattern1 - First pattern
 * @param {WeatherPattern} pattern2 - Second pattern
 * @returns {Number} Compatibility score (0-1)
 */
function calculateCompatibility(pattern1, pattern2) {
  const tempDiff = Math.abs(pattern1.temperature - pattern2.temperature);
  return 1 - tempDiff / 40;
}

// Bad: Unclear, undocumented
function calc(p1, p2) {
  return 1 - Math.abs(p1.t - p2.t) / 40;
}
```

### Vue Component Best Practices

```vue
<!-- Good: Clean, reactive Vue component -->
<template>
  <div class="weather-display">
    <h3>{{ weatherName }}</h3>
    <p>Temperature: {{ temperature }}°{{ unit }}</p>
  </div>
</template>

<script setup>
import { ref, computed, onMounted, onUnmounted } from "vue";

const props = defineProps({
  weatherSystem: {
    type: Object,
    required: true
  }
});

const weather = ref(null);

const weatherName = computed(() => {
  return weather.value?.name ?? "Unknown";
});

onMounted(() => {
  updateWeather();
});

function updateWeather() {
  weather.value = props.weatherSystem.getCurrentWeather();
}
</script>

<!-- Bad: Missing reactivity, no lifecycle management -->
<template>
  <div>{{ weather.name }}</div>
</template>

<script setup>
const props = defineProps(["weatherSystem"]);
let weather = props.weatherSystem.getCurrentWeather(); // Not reactive!
</script>
```

### Error Handling

```javascript
// Good: Handle errors gracefully
try {
  const result = await rollTable.roll();
  return parseWeather(result);
} catch (error) {
  console.error("Dynamic Weather: Error rolling table", error);
  ui.notifications.error("Failed to roll weather");
  return null;
}

// Bad: No error handling
const result = await rollTable.roll();
return parseWeather(result);
```

### Hooks Usage

```javascript
// Good: Check conditions before executing
Hooks.on("dynamicWeatherUpdate", weather => {
  if (!canvas.ready || !weather) return;
  updateScene(weather);
});

// Bad: No safety checks
Hooks.on("dynamicWeatherUpdate", weather => {
  updateScene(weather);
});
```

## API Stability

When making changes to the public API:

- **Major Version (2.0.0)**: Breaking changes allowed
- **Minor Version (1.1.0)**: New features, backward compatible
- **Patch Version (1.0.1)**: Bug fixes only

Mark deprecated features clearly:

```javascript
/**
 * @deprecated Use getCurrentWeather() instead
 */
function getWeather() {
  console.warn("getWeather() is deprecated, use getCurrentWeather()");
  return getCurrentWeather();
}
```
